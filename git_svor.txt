	cd
	cd ~
	ls
	pwd
	mkdir

	b)Change Directory To Root
	c)List Directory Contents

	e)Make Directory

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone
	git log
	git status
	git diff
	git checkout
	
	a)Downloads A Project And It's Entire Version History
	b)Lists Version History For The Current Branch
	c)Lists All New Or Modified Files To Be Committed
	d)Shows File DifferencesNot Yet Staged
	e)Switches To The Specified Branch And Updates The Working Directory
	
3. Hver er munurinn á eftirfarandi git skipunum:
	git diff 
	git diff --staged 
	git diff commit1 commit2
	
	a)Shows File Differences Not Yet Staged
	b)Shows File Differences Between Staging And The Last File Version
	c)Shows Content Differences Between Two Branches

4. Hvað er útgáfustýring (e.version control)?
	It Allows The Coder To Return To An Older Version Of The Project

5. Hverjir eru helstu kostir við að nota GIT?
	Það er Auðveldara að ná í gamlann kóða sem var t.d. síðast virkandi kóðinn
	
6. Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?
	As Often As You Save (Logic: Why Not?)

7. Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
	"Working Directory" Is Where Your Code Is Locally. "Staging Area" Is Where Your Commits Are Stored. "Repository" Is Where Your Pushed Commits Are Stored (Usually Sites Like GitHub.com)

8. Hvenær er sniðugt að nota greinar (branches)?
	When Many People Are Working On The Same Project At The Same Time To Lessen The Need For Real-Time Communication
